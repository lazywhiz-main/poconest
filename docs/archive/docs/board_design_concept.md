# ポコの巣 - ボード設計思想

> **本ドキュメントの目的** - ボードの本質的な役割、各カラムの意味付け、情報の流れを定義し、一貫した体験を提供するための指針とする。

## 1. ボードの本質的な役割

**「対話から生まれる知恵の育成場所」**

ポコの巣におけるボードは、単なる情報の保管庫ではなく、アイデアや洞察が有機的に成長していく生態系です。チャットやZoomでの対話（Flow）から生まれた思考の種が、整理・熟成されて知恵（Stock）へと発展していく過程を体現しています。

### 体験原則

- **Seamless Stock** - Flow（チャット）とStock（ボード）を意識させない自然な往復
- **One-Beat Capture** - 思いついたアイデアを即座に保存できる即時性
- **Playful Flow** - 情報整理にも遊び心を取り入れた楽しい体験

## 2. カラムの役割定義

ボードは4つのカラムで構成され、それぞれが知識の成長段階を表現しています：

### Inbox（苗床）
- **本質**: 整理されていない生の思考の一時置き場
- **特徴**: 即時性、未整理、可能性
- **主な流入源**: ユーザーによる手動保存、簡易メモ
- **状態**: ユーザーの即時判断で「これは後で見返したい」と思った生の素材

### Insights（発芽）
- **本質**: AIによって抽出された重要な洞察
- **特徴**: AI判断、自動生成、文脈理解
- **主な流入源**: AIによる会話分析、Inboxからの昇格
- **状態**: 意味解析と優先度判定を経た価値ある情報

### Themes（成長）
- **本質**: 関連する洞察をまとめたより大きなアイデア群
- **特徴**: 構造化、パターン化、知識の体系化
- **主な流入源**: 複数のInsightsの統合、プロジェクト化
- **状態**: ユーザーの目的や関心に沿って整理された知的資産

### Zoom（養分）
- **本質**: ビデオ会議からの重要な情報
- **特徴**: リッチコンテンツ、文脈保持、音声由来
- **主な流入源**: Zoom会議の録画、文字起こし、要約
- **状態**: 対面会話の記録としての特別な価値を持つ情報

## 3. 情報の流れ

アプリケーション内での情報の流れは、ユーザーの思考プロセスを自然に支援するよう設計されています：

### チャットからボードへの流入経路

1. **チャットから手動保存 → Inbox**
   - **トリガー**: ユーザーの即時判断
   - **操作方法**:
     - メッセージの左スワイプ
     - メッセージ長押しから「Inboxに保存」オプション
     - 会話中の引用（選択テキスト）保存機能
   - **特徴**: 素早い保存、最小限の情報整理

2. **AIによる自動抽出 → Insights**
   - **トリガー**: AI分析
   - **操作方法**:
     - 会話終了時や一定の長さに達した時にAIが分析
     - 定期的な会話レビュー（夜間や非アクティブ時）
     - ユーザーが「この会話から洞察を生成」と指示
   - **特徴**: 自動的な価値抽出、意味理解、優先度付け

3. **Inboxからの昇格 → Insights**
   - **トリガー**: ユーザーの精査と判断
   - **操作方法**:
     - Inboxカードの「洞察に昇格」ボタン
     - AIによる補強（タイトル生成、タグ提案、関連付け）
     - 昇格時に内容の拡充機能
   - **特徴**: 人間とAIの協働による価値の再定義

4. **Zoomからの流入**
   - **トリガー**: ビデオ会議の完了
   - **操作方法**:
     - 会議の完全記録・要約 → **Zoom**カラム
     - 会議から抽出された個別の重要ポイント → **Insights**
     - ユーザーがハイライトした瞬間 → **Inbox**
   - **特徴**: マルチモーダルなデータの統合、重要箇所の抽出

### カラム間の移動と成長

1. **Inbox → Insights**: 価値の認識と強化
   - AIによる内容の精査・拡充
   - 関連情報の自動付加
   - 優先度の判定

2. **Insights → Themes**: パターンの発見と体系化
   - 関連する複数のInsightsをグループ化
   - テーマとしての意味付け
   - 知識構造の可視化

3. **横断的活用**:
   - Zoom記録からの洞察抽出
   - テーマに基づいた新しい対話の提案
   - チャットでの過去の知見の参照

## 4. UI/UXフロー

### チャット画面の保存機能

- メッセージの横にInboxアイコン（小さな巣穴マーク）
- 長押しで「保存オプション」表示：
  - 「そのままInboxへ」（素早い保存）
  - 「洞察として保存」（AIによる拡充＋Insightsへ）
- 保存時の小さなアニメーション（ポコがメッセージを巣に運ぶ）

### AI自動抽出の通知

- AIが洞察を見つけたらバッジ通知
- ホーム画面の「今日の発見」セクションに表示
- 週次レポート「ポコの今週の発見」機能

### Inbox画面での操作

- カード右上に「昇格」ボタン
- AIによる「これは洞察になりそう」というサジェスト
- 関連する過去のInsightsの表示

### Insights画面での強化

- AIによる自動分類と整理
- 関連するInboxアイテムの表示
- 「テーマに発展させる」オプション

## 5. 実装の方向性

### 新機能の方向性

1. **知識の成長を可視化**
   - カードの進化状態の表示
   - 関連カード間のつながりの可視化
   - AIによる知識マップの自動生成

2. **自然な情報の流れ**
   - チャットからの自然な保存フロー
   - カード間の関連性の自動検出
   - コンテキストの保持と復元

3. **共同キュレーション**
   - 二人での共同編集
   - 相手の活動の可視化
   - 共感や反応の表現

4. **知識の活用**
   - 関連する過去の対話の提案
   - テーマに基づく新しい対話の提案
   - 知識の再利用と発展

### 実装優先度

1. **まず実装すべき機能**:
   - チャットからInboxへの手動保存（基本的なフロー）
   - Inboxカードの基本的な管理（編集、削除、移動）
   - シンプルなAI洞察抽出（会話の要約点）

2. **次のステップ**:
   - Inbox → Insights昇格機能
   - AIによる洞察の自動抽出強化
   - 関連性の可視化

3. **発展機能**:
   - チャット内の過去のInbox/Insights参照
   - テーマへの発展アシスト
   - Zoom統合の強化

## 6. 成功指標

1. **ユーザー体験指標**:
   - チャットからボードへの保存アクション完了時間 ≤ 1秒
   - カラム間のカード移動の直感性（エラー率の低さ）
   - 情報の再利用頻度

2. **価値提供指標**:
   - 生成されたInsightsの週間数
   - Themesへの昇格率
   - チャット中の過去の知見参照頻度

3. **技術指標**:
   - AIによる洞察抽出の精度
   - 関連性検出の適合性
   - 検索/フィルタリングの応答時間

## 7. 将来の拡張可能性

1. **パーソナライズされた知識ネットワーク**
   - 個人の関心領域に基づく自動分類
   - 長期的な知識の成長トラッキング

2. **外部サービス連携**
   - NoitonやObsidianなどの知識管理ツールとの連携
   - カレンダーとの連動による時間軸ナビゲーション

3. **コミュニティ拡張**
   - 複数ユーザー間での知見共有
   - テーマごとの小規模コミュニティ形成

---

この設計思想に基づき、ポコの巣のボード機能は「手軽な保存」から始まり「価値ある知識」へと進化する体験を提供します。ユーザーにとっては「保存したいものはすぐInboxへ」「AIが見つけた価値はInsightsで確認」という明確な精神モデルが形成され、情報整理の負担を減らしつつ知的資産を蓄積できるようになります。
