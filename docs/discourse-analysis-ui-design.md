# ディスクール分析機能 - UI設計と体験フロー

## 設計原則

### 1. 深層探求のためのUI
- **表面的な情報を隠し、深層を浮かび上がらせる**インターフェース
- **直感的な発見を促す**ビジュアル表現
- **分析者の思考を邪魔しない**ミニマルなデザイン

### 2. 段階的な深掘り体験
- **概要 → 詳細 → 深層分析**の自然な流れ
- **仮説形成 → 検証 → 洞察発見**のサイクル支援
- **複数の視点**からの同時分析可能性

## 4. 分析結果配置の設計思想

### A. タブ式配置の優位性

**1. 認知負荷の軽減**
- 一度に一つの分析結果に集中できる
- 情報過多による分析品質の低下を防止
- 各分析の独自性を保持

**2. 段階的理解の促進**
```
感情分析 → パターン分析 → ディスクール分析 → 統合洞察
（表層）   （中層）     （深層）         （統合）
```

**3. 結果の保存と比較**
- 各分析結果を独立して保存・編集可能
- 過去の分析との比較が容易
- セッション間の変化追跡が可能

### B. レスポンシブ対応
```css
/* デスクトップ: 横幅フル活用 */
.analysis-results-desktop {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

/* タブレット: コンパクト表示 */
.analysis-results-tablet {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

/* モバイル: 最小限の情報に絞る */
.analysis-results-mobile {
  display: flex;
  flex-direction: column;
  gap: 8px;
}
```

この**新しいフロー設計**により、ユーザーが求める「分析選択→実行→結果確認」の体験が実現できます。特に**タブ式分析結果表示**により、複数の分析結果を効率的に管理・比較できる仕組みになっています。
