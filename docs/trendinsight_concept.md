# Design Radar - プロダクト要求仕様書

## プロジェクトの目的

**「コンセプトの転換・カテゴリー破壊・哲学的プライシング・体験の劇的な変化」を持つ製品を、自動的に発見・調査・蓄積し、パターンからインサイトを生成するシステム**

---

## コアコンセプト

### 人間がやるべきこと
- 気になる製品を見つけたら「もっと調べて」とトリガー
- 週1回、新着をざっと確認（10分）
- 月1回、インサイトレポートを読む（15分）

### システムがやるべきこと
- RSS情報源から毎日自動収集
- 各製品を基準でスコアリング
- スコアが高いものは自動で深掘り調査
- データを構造化して蓄積
- 定期的にパターンを分析してインサイト生成

---

## Phase 1: 自動収集と初期判定

### 情報源
- Dezeen (プロダクトデザインフィード)
- Yanko Design
- Red Dot Award受賞作（年1回）
- iF Design Award受賞作（年1回）
- その他、後から追加できる設計

### 毎日の自動処理フロー

```
1. RSS/情報源から新着記事を取得
   ↓
2. 各記事について、AIが以下を実行:
   - 元記事を読む
   - 日本語で100-150文字のサマリー生成
   - 4つの基準で0-10点スコアリング:
     a) コンセプトの転換度
     b) カテゴリー破壊度
     c) 哲学的プライシング
     d) 体験の劇的な変化度
   - 合計スコアを算出（最大40点）
   - 該当する場合、その理由を簡潔に説明
   ↓
3. データベースに保存
   - 製品タイトル（原文 + 日本語）
   - URL
   - サマリー
   - スコア（各基準 + 合計）
   - 該当理由
   - ステータス: 「新着」
   - 発見日時
```

### スコアリングの基準

**参考となる「該当する」例**:
- Teenage Engineering EP-1320（中世楽器）→ コンセプト転換 9/10
- Ventete 折りたたみヘルメット → カテゴリー破壊 9/10
- Balenciaga $4,000テープ → 哲学的プライシング 10/10
- Onスプレー靴 → 体験の劇的変化 8/10

**参考となる「該当しない」例**:
- 掃除機の吸引力が20%向上 → スペック改善（スコア2-3点）
- ノートPCのパーツ交換可能に → 機能追加（スコア3-4点）
- イヤホンの音質向上 → 性能改善（スコア2-3点）

## Phase 2: 段階的な深掘り調査

### 自動トリガー条件

**Level 1調査（基本情報収集）**
- 条件: スコア7点以上
- 実行タイミング: 記事発見から30分以内
- 調査内容:
  - ブランド/デザイナーの公式情報
  - 価格・発売時期
  - 基本スペック
  - 主要メディアの反応（2-3件）
- 結果: データベースに「調査結果Level 1」として追加

**Level 2調査（文脈と背景）**
- 条件: スコア8点以上
- 実行タイミング: Level 1完了から1時間後
- 調査内容:
  - デザイナー/ブランドの過去作品
  - 類似製品の存在（過去5年）
  - 業界での位置づけ
  - SNS/メディアでの評価トレンド
- 結果: 「調査結果Level 2」として追加

**Level 3調査（深層分析）**
- 条件: スコア9点以上 OR ユーザーが手動トリガー
- 実行タイミング: Level 2完了後 OR 即時（手動時）
- 調査内容:
  - デザイン哲学の革新性分析
  - 文化的・社会的文脈での意味
  - 5年後のインパクト予測
  - 歴史的に類似した製品事例
  - なぜ「今」このタイミングで登場したか
- 結果: 「調査結果Level 3」として追加

### 調査の進行状態

製品のステータスは自動で遷移:
```
新着 → 調査中(L1) → 調査中(L2) → 調査中(L3) → 完了
       ↓             ↓             ↓
    (スコア6以下)  (スコア7以下)  (スコア8以下)
       ↓             ↓             ↓
      終了          完了          完了
```

## システムが満たすべき品質要求

### パフォーマンス
- ダッシュボードの初期表示: 2秒以内
- 製品詳細ページの表示: 1秒以内
- 手動調査トリガー → 結果表示: 3分以内

### 信頼性
- RSS収集の失敗時は自動リトライ（3回まで）
- AI分析の失敗時はエラーログ + 人間による確認フラグ
- データベースバックアップ: 日次自動

### 拡張性
- 新しいRSS情報源を追加: 設定ファイルに1行追加するだけ
- 新しいスコアリング基準を追加: AI プロンプト修正のみ
- 新しい調査レベルを追加: 構造化されたコードで対応可能

### セキュリティ
- API Keyは環境変数で管理
- データベースアクセスは認証必須（将来的に複数ユーザー対応時）

## 成功の定義

このシステムが成功したと言えるのは:

1. **毎週10分の確認だけで、面白い製品を見逃さない**
2. **「これ調べて」と言えば、30分後には詳細レポートができている**
3. **3ヶ月後、自分の興味のパターンが明確に見えている**
4. **6ヶ月後、「次に来そうなトレンド」が予測できている**
